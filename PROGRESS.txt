Stage 2 Local GPU Scaling
- Additional local WSL CUDA batch-size probes were run after the initial smoke test.
- Outcomes:
  - `--batch-size 4096`: crashed the local system
  - `--batch-size 2048`: crashed the local system
  - `--batch-size 1024`: passed both a short probe and a longer probe
- Important paper-vs-current distinction:
  - paper literal batch size: `4096`
  - local physical batch size currently validated: `1024`
  - before the next code change, local training at `1024` did **not** mimic the paper batch size; the optimizer still stepped every `1024` examples
- Trainer update:
  - `train_stage2_transformer.py` now supports `--gradient-accumulation-steps`
  - this allows a local run such as `--batch-size 1024 --gradient-accumulation-steps 4`
  - effective batch size of that run: `4096`
  - physical microbatch size remains `1024`
  - this is the correct way to mimic the paper batch size on the current laptop GPU without fitting all `4096` examples in VRAM at once
